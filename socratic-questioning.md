# Socratic Questioning Framework for PRDs

Use 3-5 questions to sharpen feature thinking before drafting. Pick the most relevant from each category.

## 1. Problem Clarity
- **What specific user pain point does this solve?** (Look for concrete examples, not abstractions)
- **How do we know this is a real problem?** (Evidence: interviews, support tickets, data)
- **Who experiences this most acutely?**
- **What's the cost of NOT solving this?**

## 2. Solution Validation
- **Why is this the right solution for that problem?**
- **What alternatives did you consider? Why reject them?**
- **What's the simplest version that solves the core problem?**
- **How will users discover this feature?**

## 3. Success Criteria
- **How will we know if this is successful?** (Specific, measurable)
- **What would make this a failure?**
- **What metric are we trying to move? By how much?**

## 4. Constraints & Trade-offs
- **What are the technical constraints or risks?**
- **What are we NOT doing as part of this?**
- **If we had half the time, what would we cut?**

## 5. Strategic Fit
- **Why build this NOW vs 6 months from now?**
- **How does this fit our broader product strategy?**
- **How does this affect our competitive position?**

## Flow
Start broad (problem clarity) → narrow (solution, success) → zoom out (strategic fit).

## Red Flags to Watch For
- Vague language: "users want better..." → What specifically?
- Solution-first thinking: can describe feature but not the problem
- No evidence: "I think users would like..." → How do we know?
- Unclear success: can't articulate what success looks like
